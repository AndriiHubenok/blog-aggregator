
# Blog Aggregator (Gator)

A command-line interface (CLI) RSS feed aggregator built in Go. This tool allows you to register users, manage and follow RSS feeds, scrape feeds continuously in the background, and browse recent posts directly from your terminal.

## Features
* **User Management**: Register and log in as different users. 
* **Feed Management**: Add new RSS feeds, view available feeds, and follow/unfollow specific feeds.
* **Background Scraping**: Run a continuous aggregation worker to fetch and store new posts from your feeds at a custom time interval.
* **Post Browsing**: Browse the latest posts from the feeds you are subscribed to.
* **Type-safe Database**: Powered by PostgreSQL with type-safe database queries generated by SQLC.

## Prerequisites
* [Go](https://golang.org/) (v1.26.2 or compatible)
* [PostgreSQL](https://www.postgresql.org/)
* [SQLC](https://sqlc.dev/) (for generating database code)
* [Goose](https://github.com/pressly/goose) (for database migrations)

## Setup and Installation

1. **Install dependencies:**
   ```
   go mod tidy

2. **Set up the Database:**
Ensure PostgreSQL is running locally and create a database named `gator`.
The default connection string used in the application is:
```text
postgres://postgres:postgres@localhost:5432/gator

```


3. **Run Database Migrations:**
Navigate to the `sql/schema` directory and run goose to apply the schema (which includes tables for `users`, `feeds`, `feed_follows`, and `posts`):
```bash
cd sql/schema
goose postgres "postgres://postgres:postgres@localhost:5432/gator" up
cd ../..

```


4. **Generate SQL Queries:**
Because the `internal/database` directory is excluded from version control via `.gitignore`, you must generate the Go database code using sqlc before building:
```bash
sqlc generate

```


5. **Build the Application:**
```bash
go build -o gator

```



## Usage

Run the CLI tool using the generated binary (e.g., `./gator <command> [arguments]`). The app automatically stores your active user session in a `.gatorconfig.json` file in your home directory.

### User Management

* **Register a new user:**
```bash
./gator register <username>

```


* **Log in as an existing user:**
```bash
./gator login <username>

```


* **List all users:**
```bash
./gator users

```


* **Reset (Delete all users):**
```bash
./gator reset

```



### Feed Management

* **Add a new RSS feed:**
```bash
./gator addfeed <name> <url>

```


* **List all available feeds:**
```bash
./gator feeds

```


* **Follow a feed:**
```bash
./gator follow <url>

```


* **Unfollow a feed:**
```bash
./gator unfollow <url>

```


* **List all feeds you are following:**
```bash
./gator following

```



### Aggregation and Browsing

* **Start the aggregator (scraping worker):**
Collects posts from the next available feed at a specified time interval (e.g., `1m`, `30s`, `1h`). This process runs continuously until you stop it.
```bash
./gator agg <time_duration>

```


* **Browse posts:**
View the latest posts from the feeds you follow. You can optionally pass a limit to dictate how many posts to return (the default is 2).
```bash
./gator browse [limit]

```



## Project Structure

* `main.go` & `handler.go`: Contain the CLI state, command registry, and execution handlers.
* `rss.go`: Handles fetching XML from RSS feeds and parsing them into Go structs.
* `internal/config/`: Manages reading and writing user state to `.gatorconfig.json`.
* `internal/database/`: Auto-generated type-safe database queries and models (generated by SQLC).
* `sql/schema/`: SQL migration files defining the database structure.
* `sql/queries/`: Raw SQL queries used by SQLC to generate the `internal/database` package.
